# Creating Content

Learn how to add new pages, organize your documentation, and leverage Nextra's powerful features.

## Adding New Pages

### Creating MDX Files

Create new documentation pages by adding MDX files to the `content/` directory:

```mdx

This is a new documentation page with **markdown** support.

## Features

- Full markdown support
- React components
- Syntax highlighting
- And much more!
```

## Navigation Configuration

### Main Navigation

Update `content/_meta.js` to add pages to navigation:

```javascript
export default {
  index: "Getting Started",
  "quick-start": "Quick Start",
  "guides": "User Guides",
  "api": "API Reference",
  "troubleshooting": "Troubleshooting"
}
```

### Nested Navigation

For subdirectories, create `_meta.json` files:

```json
{
  "installation": "Installation Guide",
  "configuration": "Configuration",
  "advanced": "Advanced Topics"
}
```

#### When to Use Each Approach

**Use Markdown Tables when:**
- Creating simple, straightforward tables
- Content is primarily text-based
- You want to keep the source readable
- Quick documentation needs

**Use Table Component when:**
- You need more styling control
- Tables contain complex content or components
- You want consistent styling across your site
- Building data-heavy documentation

### Images and Media

Add images and media files to your documentation. All images should be placed in the `/public` directory of your project.

<Callout type="info">
  **Image Root Directory**: The `/public` folder is the root directory for serving static files. When you reference `/images/logo.png` in your MDX, it actually points to `/public/images/logo.png` in your project structure.
</Callout>

## Content Best Practices

### Writing Guidelines

1. **Clear headings** - Use descriptive, hierarchical headings
2. **Concise paragraphs** - Keep paragraphs short and focused
3. **Active voice** - Use active voice for clarity
4. **Consistent terminology** - Use consistent terms throughout
5. **Code examples** - Include practical code examples

### Structure Guidelines

1. **Logical flow** - Organize content in logical progression
2. **Cross-references** - Link related content
3. **Table of contents** - Use headings for automatic TOC
4. **Scannable format** - Use lists, tables, and callouts

### SEO Optimization

1. **Descriptive titles** - Use clear, descriptive page titles
2. **Meta descriptions** - Add page descriptions
3. **Heading hierarchy** - Use proper heading structure (H1, H2, H3)
4. **Internal linking** - Link between related pages
